How It Works

A 3BLD solution decomposes the 12 edges and 8 corners into independent permutation–orientation structures. This tool exhaustively enumerates every cycle configuration and precomputes how many distinct scrambles realise it, giving exact probabilities for any filter.

Example

Consider the scramble U2 L2 U' R D2 F' L D B' D R2 F2 L2 F2 U' R2 U B2 U2 B2 U. With buffer UF (edges) and UFR (corners), the cycle decomposition is:

Edges:

$$\underbrace{(\mathrm{UF},\,\mathrm{BU},\,\mathrm{DL},\,\mathrm{FL})}_{o=0,\;\text{buffer}}\;\underbrace{(\mathrm{UR},\,\mathrm{RB})}_{o=0}\;\underbrace{(\mathrm{UL},\,\mathrm{LB})}_{o=0}\;\underbrace{(\mathrm{FR},\,\mathrm{BD})}_{o=1}\;\underbrace{(\mathrm{DR})}_{o=1}\;\underbrace{(\mathrm{DF})}_{o=0,\;\text{solved}}$$

Corners:

$$\underbrace{(\mathrm{UFR},\,\mathrm{LDF},\,\mathrm{LUB},\,\mathrm{BDR},\,\mathrm{DBL},\,\mathrm{RUB},\,\mathrm{RDF})}_{o=2,\;\text{buffer}}\;\underbrace{(\mathrm{UFL})}_{o=1}$$

Here $o$ is the net orientation of a cycle: $o = \bigl(\sum_{\text{cycle}} \mathrm{ori}(i)\bigr) \bmod m$, where $m=2$ for edges and $m=3$ for corners. Orientation values: $0$ = correct, $1$ = flipped (edges) or CW-twisted (corners), $2$ = CCW-twisted (corners). A 3-cycle with $o=0$ is orientation-clean and can be solved by a single 3-style alg. The global constraint $\sum_{\text{all}} \mathrm{ori}(i) \equiv 0\pmod{m}$ always holds.

Cycle Configuration

A cycle configuration is described by the buffer cycle $(p_0, o_0)$ and a list of secondary cycle types $(p_j, o_j, r_j)$, where $p_j$ is the cycle length, $o_j$ the net orientation, and $r_j$ the number of cycles of that type. For this scramble:

Piece typeBuffer $(p_0,o_0)$Secondary types $(p_j,o_j,r_j)$
Edges$(4,0)$$(2,0,2),\;(2,1,1),\;(1,1,1),\;(1,0,1)$
Corners$(7,2)$$(1,1,1)$

Counting Scrambles per Configuration

$$\operatorname{count} = \frac{(P-1)!\cdot O^{P-1-\sum_j r_j}}{\displaystyle\prod_j p_j^{r_j} \cdot r_j!}$$

For this scramble ($P=12,\,O=2$ for edges; $P=8,\,O=3$ for corners):

$$\operatorname{count}_{\mathrm{edges}} = \frac{11!\cdot 2^{12-1-5}}{2^2\cdot2!\;\cdot\;2^1\cdot1!\;\cdot\;1^1\cdot1!\;\cdot\;1^1\cdot1!} = \frac{39{,}916{,}800\cdot 64}{16} = 159{,}667{,}200$$ $$\operatorname{count}_{\mathrm{corners}} = \frac{7!\cdot 3^{8-1-1}}{1^1\cdot1!} = 5040\cdot 729 = 3{,}674{,}160$$

Alg Count Estimation

$$\text{algs} = \text{algs}_{\text{base}} + \text{algs}_{\text{flip/twist}} - \text{algs}_{\text{float3}}$$
  • $\text{algs}_{\text{base}}$ $= \frac{1}{2}\left[ (p_0 - 1) + \sum_{\substack{j \\ p_j > 1}} r_j(p_j + 1) \right]$: Each secondary cycle with $p_j > 1$ contributes $p_j + 1$ letters — $p_j$ for its pieces plus $1$ for the cycle-break letter.
  • $\text{algs}_{\text{flip/twist}}$: additional algs for misoriented in-place pieces ($p_j=1,\,o_j > 0$). Edges: $\lceil\mathrm{flips}/4\rceil$. Corners with $k_1$ CW-twisted and $k_2$ CCW-twisted: $\lfloor k_1/3\rfloor+\lfloor k_2/3\rfloor+\lceil((k_1\bmod 3)+(k_2\bmod 3))/3\rceil$.
  • $\text{algs}_{\text{float3}}$: number of orientation-clean secondary 3-cycles ($p_j=3,\,o_j=0$), each solvable by a single commutator — saves 1 alg each.

Parity

$$\text{parity} = (2 \cdot \text{algs}_{\text{base}}) \bmod 2$$

Any valid 3BLD scramble has equal edge and corner permutation parity, so filtering one automatically filters both.

Sampling

Each valid cycle configuration receives a weight equal to its scramble count. Two CDFs are built — even-parity pool and odd-parity pool — weighted by count. To generate a scramble: pick parity in proportion to pool totals; sample a configuration from the CDF; instantiate a random permutation and orientation assignment matching that configuration; solve with min2phase.
