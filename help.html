BLD Master generates cube scrambles based on specific criteria you define. Instead of random scrambles, you can practice specific scenarios like edge cycles, corner twists, parity cases, and more.

ðŸ”§ Basic Usage

Edge & Corner Input Fields: Enter JavaScript boolean expressions to filter scrambles. Leave empty to allow any case. Use these variables in your filter expressions:

  • parity - Cube parity (0 = even, 1 = odd).
  • cycles - Number of cycle breaks.
  • algs - Estimated alg count (2 letters 1 alg). A float 3-cycle / corner 3-twist / edge 4-flip is counted as 1 alg. Parity is counted as 0.5 algs in both edge and corner.
  • float1 - Number of solved edges/corners.
  • bad1 - Number of flipped edges/twisted corners.
  • float2,float3,float4,float5 - Number of 2,3,4,5-cycles with correct orientation.
  • bad2,bad3,bad4,bad5 - Number of 2,3,4,5-cycles with wrong orientation.
