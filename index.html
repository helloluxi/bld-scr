3BLD Scrambler

What is this?

3BLD Scrambler generates 3x3 scrambles filtered by the cycle structure of their BLD solution. Buffers are UF (edges) and UFR (corners). Scramble in your own orientation.

This project is actually a vibe-coded remastering of another tool when I just started learning programming and wanted to make something fun. It just works. While I can't think of practical use case of this project, I would self-promote another tool of mine — LuTimer for more serious training.

Concepts

  • Parity: even / odd, whether the scramble has BLD parity.
  • Flips / Twists: number of flipped edges or twisted corners, buffer excluded.
  • Cycle Breaks: how many times the memo must jump to a new starting piece.
  • Float 3-style: number of floating 3-cycles (orientation-clean 3-cycles, buffer excluded).
  • Algs: estimated algorithm count (each 3-style pair = 1 alg; a float-3 / 3-twist / 4-flip saves 1; parity adds 0.5).

The Probability line shows what fraction of all scrambles match your current filter — useful to gauge how rare your target case is. Hitting Scramble! updates the URL so you can bookmark or share your current setup.

Help

Write a JavaScript boolean expression directly. The expression is evaluated for each possible cycle structure; only scrambles whose structure satisfies both the edge and corner conditions are generated.

Available variables — all integers unless noted:

  • parity — 0 (even) or 1 (odd).
  • breaks — number of cycle breaks, i.e. how many times the memo had to jump to a new starting piece.
  • algs — estimated algorithm count: each 3-style pair costs 1 alg (so each piece costs 0.5); a floating 3-cycle / 3-twist group / 4-flip saves 1 alg; parity adds 0.5.
  • closed1 — solved pieces in-place with correct orientation (free, no alg needed).
  • open1 — pieces in-place but misoriented: flipped edges or twisted corners.
  • closed2 … closed5 — clean k-cycles (correct orientation) that appear as secondary cycles.
  • open2 … open5 — k-cycles with at least one misoriented piece.

Your last 5 expressions are saved per-field. Use ↑↓ in the input to navigate history. Hitting Enter generates scrambles. The URL updates so you can bookmark or share a filter.
