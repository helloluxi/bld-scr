3BLD Scrambler

What is this?

3BLD Scrambler generates 3x3 scrambles filtered by the cycle structure of their BLD solution. Buffers are UF (edges) and UFR (corners) only for now. Scramble in your own orientation.

I would also like to self-promote another tool of mine — BLD Doctor for more serious training.

Concepts

  • Parity: even / odd, whether the scramble has BLD parity.
  • Flips / Twists: number of flipped edges or twisted corners, buffer excluded.
  • Cycle Breaks: how many times the memo must jump to a new starting piece.
  • Float 3-style: number of floating 3-cycles (orientation-clean 3-cycles, buffer excluded).
  • Algs: estimated algorithm count (each 3-style pair = 1 alg; a float-3 / 3-twist / 4-flip saves 1; parity adds 0.5).

The Probability line shows what fraction of all scrambles match your current filter — useful to gauge how rare your target case is. Hitting Scramble! updates the URL so you can bookmark or share your current setup.

The Basic panel's Algs slider counts under Basic 3-style. For Floating plain 3-cycles only, Full Floating, or Full Floating Parity counts, use the Advanced panel and write expressions over alg, alg-closed3, algFF, or algFFP.

Help

Write a JavaScript boolean expression directly. The expression is evaluated for each possible cycle structure; only scrambles whose structure satisfies both the edge and corner conditions are generated.

Available variables — all integers unless noted:

  • parity — 0 (even) or 1 (odd).
  • breaks — number of cycle breaks, i.e. how many times the memo had to jump to a new starting piece.
  • algs — alg count under Floating plain 3-cycles only. For Basic 3-style use alg; for Full Floating use algFF; for Full Floating Parity use algFFP.
  • closed1 — solved pieces in-place with correct orientation (free, no alg needed).
  • open1 — pieces in-place but misoriented: flipped edges or twisted corners.
  • closed2 closed3 — clean k-cycles (correct orientation) that appear as secondary cycles.
  • open2 open3 — k-cycles with at least one misoriented piece.
  • cwTwist / ccwTwist — number of CW / CCW twisted-in-place corners. Corner-only; for edges, use open1 instead.

Your last 5 expressions are saved per-field. Use ↑↓ in the input to navigate history. Hitting Enter generates scrambles. The URL updates so you can bookmark or share a filter.

Examples

Floating plain 3-cycle drill

Edge: closed3>0

Cases where full floating saves

Edge: alg>algFF Corner: alg>algFF

4-flip practice (edges)

Edge: open1>=3

Three or more flipped-in-place edges forces a 4-flip alg.

3-twist practice (corners)

Corner: cwTwist>=3 || ccwTwist>=3 || (cwTwist==2 && ccwTwist==0) || (ccwTwist==2 && cwTwist==0)

Forces a 3-twist alg case. Two same-direction twists invoke a 3-twist; mixed CW+CCW pairs cancel into a 2-twist.

Non-buffer 2-twist (corners)

Corner: cwTwist==1 && ccwTwist==1

LTCT

Corner: parity==1 && open1>=1

Odd parity with a twisted-in-place corner — the canonical LTCT setup.

T2C

Corner: parity==1 && open2>=1

Easy scrambles

Edge: algs<=5 Corner: algs<=3
